## 0. 왜 지금 이 안건인가

오피스아워 1순위 지적: **"What's New / Why You / Value가 불명확. 기능을 섞어서 핵심이 안 보인다."**
처방: **목적(Y) 하나로 정렬, 안 맞는 건 걷어내라.** + **3인이 각자 Y-tree 그려 정렬 확인.**

→ 이 문서는 계휴의 Y-tree 결과 + 제안. **승원·윤지도 각자 Y-tree를 그린 뒤** 비교하자 (§6 프롬프트).

## 1. 제안하는 코어

**코어 감정 = 고양감(uplift)** — 책 한 문장이 나를 끌어올리는 순간.
독서의 진짜 보상은 완독·분량·스트릭이 아니라 그 순간인데, 늘 휘발된다.
→ **붙잡고(한 문장+코멘트) · 증폭하고(LLM이 질문) · 쌓는다(내 서재).**

## 2. 피치 4문장 (제안 초안)

| | 초안 |
|---|---|
| **What's New** | "혼자 읽어도, 한 문장을 남기면 '왜 그게 좋았어?'라고 물어봐 주는 — 언제나 열려 있는 1인 독서모임" |
| **What's Different** (vs 북모리) | 북모리는 문장을 *저장*한다. ReadingGo는 문장으로 *대화*한다 (북모리 AI 없음 확인) |
| **Why You / 해자** | 콘텐츠(LLM에 다 있음)가 아니라 **내 축적 맥락**(문장·왜·이력)에 기반한 관계 = 인터랙션 해자. 쓸수록 나를 더 아는 진행자 |
| **Value** | 휘발되던 한 줄이 → 다시 읽고 싶게 만드는 나의 고양 기록 |

### "그냥 ChatGPT 아냐?" 방어
먼저 물어봄 (안 시켜도) · 쌓임 (휘발 아님) · 읽는 흐름 안 (내 문장이 재료)

## 3. 코어 루프

```
한 문장 저장 → 참새 "왜 이 문장 남겼어요?" → 답(또는 스킵) → (선택) 후속 질문 → 대화 축적
```

## 4. 스코프 제안 (무엇이 코어, 무엇이 받침)

| | 항목 |
|---|---|
| **코어 (남은 2주 화력 집중)** | 한 문장 + LLM 파트너(왜 남겼어) + 내 서재(고양 기록) |
| **받침 / 후순위 (살리되 고도화 X)** | 소셜 · 마을 · 게임화(리그) |
| **AI-네이티브 단일 포인트** | 이해밀도 — "왜 남겼어" 질문 (OCR·STT·요약은 비목표) |

> ⚠️ **민감**: "소셜/마을 후순위"는 윤지의 마을 작업에 닿음. **지우는 게 아니라**, 남은 2주를 코어에 모으자는 제안. 무엇이 코어/받침인지 **같이** 정하자.

## 5. 이미 정렬된 것 (오피스아워가 칭찬/검증)

- ✅ 한 문장 입력 '선택'화 (진입 마찰↓)
- ✅ 둥지 무너짐(채찍) 제거 — 단, 고양감을 '새로운 방식'으로 줄 것 (숙제)
- ✅ PostHog 정량 리텐션 토대 (#296)
- ✅ 게스트 우선 온보딩 (#298)

## 6. 팀 정렬 절차 (제안)

1. **승원·윤지: 이 문서 §1~2 결론을 보기 전에**, 각자 `/whytree`로 본인 Y-tree 먼저 그린다 (편향 방지).
2. 셋의 Y-tree를 펼쳐 비교.
   - 같은 곳으로 모이면 → 강력한 정렬 + 피치 스토리
   - 갈라지면 → **지금 해야 할 진짜 대화** (피치 당일보다 천 배 낫다)
3. 그 다음 이 문서 + #303 보고 반론·보완.
