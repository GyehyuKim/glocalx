# 기프타로 MVP 스펙 v1

> 기준일: 2026-05-12  
> 상태: 팀 공유 / 구현 착수 기준

---

## 핵심 결정

**기프타로가 소유하는 순간은 "받는 순간"이다.**

발신자가 선물을 보내는 행위 자체는 이미 의미를 담고 있다.  
기프타로는 **받는 사람이 카드를 열어보는 순간**에 집중한다.  
그 순간에 "이 사람이 나를 이 정도로 생각했구나"를 느끼게 하는 것이 우리의 역할이다.

MVP에서 결제는 제거한다. 상품은 카카오 선물하기 등 기존 채널로 아웃한다.

---

## 제품 구조

```
[발신자] 기프타로 입력 플로우
    → 의미 카드 + 상품 추천 생성
    → 공유 링크 생성
         ↓ 카카오톡으로 링크 전달
[받는 사람] 링크 열기
    → 의미 카드 확인
    → 상품 정보 확인 (가격 미노출)
    → 공유
```

두 개의 페이지로 구성한다.

- `sender` — 발신자 입력 플로우
- `receive` — 받는 사람 랜딩

백엔드 없이 순수 HTML/JS로 구현한다.  
발신자가 입력한 데이터는 URL 파라미터에 인코딩해서 링크에 담는다.

---

## Page 1 — 발신자 플로우 (`sender`)

### 목표

3스텝 안에 링크를 만들어 복사할 수 있어야 한다.

### 스텝 구성

| 스텝 | 내용 |
|------|------|
| Step 1 | 받는 사람 정보 — 이름, 별자리(또는 생년월일), 관계 |
| Step 2 | 빌어주고 싶은 운 + 예산 |
| Step 3 | 결과 확인 (의미 카드 미리보기 + 상품 추천) → 링크 복사 |

### MVP 범위

- 정합성: 입력값에 따라 의미 카드 내용과 상품 카테고리가 실제로 달라져야 함 (하드코딩 금지)
- 링크 생성: "링크 복사" 버튼 하나로 완결
- 카드 뽑기 인터랙션 (타로 카드 덱): Step 2에서 통합하거나 제거 검토 — 지금은 스텝 수 줄이는 게 우선
- 결제, 패키지 선택, checkout: MVP 제외

---

## Page 2 — 받는 사람 랜딩 (`receive`)

### 목표

링크를 열었을 때, "나를 위해 만든 것"이라는 감각을 주고 공유로 이어진다.

### 화면 구성 (순서대로)

**1. 인트로**
- "OO님이 보낸 카드가 있어요"
- 발신자 이름 표시

**2. 의미 카드**
- 개인화된 해석 메시지 (URL 파라미터에서 렌더링)
- 별자리 / 오행 등 시각 요소

**3. 상품 정보**
- 추천 상품명 + 간단한 설명
- **가격 미노출**
- 상품 링크 (카카오 선물하기 등 외부 채널)

**4. 공유 섹션**
- `나도 해보기` 버튼 → sender 페이지로 이동 (바이럴 루프)
- `링크 공유하기` 버튼 → Web Share API (모바일 네이티브 공유)
- `링크 복사하기` 버튼 → 클립보드 복사

### MVP 제외 항목

- 카드를 "여는" 인터랙션 (봉투 열기 등) — 이후 검토
- 받는 사람 반응 수집 (고마워요 / 센스 있어요 등) — 이후 검토
- 로그인 / 사용자 계정

## 기술 접근

- 단일 HTML 파일 또는 `sender.html` + `receive.html` 분리
- 데이터 전달: URL search params (`?name=...&star=...&msg=...`)
- 백엔드 없음 — GitHub Pages 정적 배포 유지
- 의미 카드 생성: JS 내부 룰 테이블 (별자리/일간 × 관계 × 운 → 메시지 템플릿 + 상품 카테고리)

## 현재 데모와의 차이

| 항목 | 현재 데모 | MVP 스펙 |
|------|---------|---------|
| 스텝 수 | 9스텝 | 3스텝 |
| 결과 정합성 | 하드코딩 | 입력 기반 동적 생성 |
| 받는 사람 페이지 | 없음 | 신규 구현 |
| 결제 | 있음 (mock) | 제거 |
| 공유 | 없음 | 3종 버튼 |
| 상품 가격 | 노출 | 미노출 |
