# GosiOps — 고시원 AI 계약 관리

> 원장님이 분쟁 케이스를 보내면, 계약서가 스스로 진화한다.

---

---

## 문제

한국 고시원 원장님(약 4만 개 사업장)은 입실 계약을 아날로그로 처리합니다.

```
현재 방식:
워드/한글로 계약서 작성 → 프린트 → 대면 서명
분쟁 발생 → 카카오톡으로 문의 → 수동 수정 → 재프린트
```

실제로 확인한 것:
- 손원장님이 계약서 수정을 직접 요청 (행동으로 증명된 수요)
- 동료 원장님 2명도 같은 문제 보유
- 원장 향 관리 툴: 한국 시장에 전무 (임차인 향 플랫폼만 존재)

---

## AI가 어디에 들어가는가

```
[기존]  원장님 경험 → 머릿속에서 사라짐

[GosiOps]
원장님 경험/분쟁 케이스
        ↓
   LLM이 조항으로 변환
        ↓
  계약서 자동 업데이트
        ↓
  네트워크 원장님 전체 적용
```

실제 이번 인터뷰에서 수집한 케이스 → AI가 조항으로 만든 예시:

| 분쟁 케이스 | AI 생성 조항 |
|---|---|
| 공용 식기를 방으로 가져감 | "주방 비품 개인 실 반입 금지, 위반 시 경고" |
| 밀프렙으로 공용 밥솥 독점 | "1인 1식 초과 대량 조리 금지" |
| 친구를 방으로 불러들임 | "외부인 시설 내 방문 전면 금지" |
| 이틀 전 갑자기 퇴실 통보 | "15일 전 서면 통보 의무, 미달 시 보증금 공제" |

AI가 분쟁 데이터를 학습할수록 계약서가 강해집니다. 이게 경쟁자가 복제할 수 없는 moat입니다.

---

## 수요 증거 (인터뷰 요약)

**1차: 계약서 수정 요청 (행동 증거)**
- 손원장님이 직접 계약서 작성/수정 도움 요청
- 지인 원장님 2명(이예린님, 장계련님) 버전도 추가 요청
- → 이미 같은 문제를 가진 네트워크 존재 확인

**2차: 현장 인터뷰 (2026-05-12)**
- 30-40방 중 외국인 10명 (영국, 프랑스, 이탈리아, 말레이시아, 키르기스스탄 등)
- 외국인 입실자가 검증된 고시원을 지인 외국인에게 소개 → 자연 레퍼럴 발생
- 분쟁 케이스 4건 수집 (위 표 참고)
- 계약서 관련 추가 수요 직접 표현

---

## 솔루션

**Stage 1 (지금 ~ 4주): 에이전시 — 소프트웨어 없음**

```
원장님 카카오톡으로 입실자 정보 전달
        ↓
GosiOps가 AI로 계약서 작성 (한/영)
        ↓
링크로 전달 → 입실자 서명
        ↓
PDF 저장
```

방당 X원/월 (30방 기준 월 3만원 수준)

**Stage 2 (4-16주): SaaS MVP**
- 원장님 대시보드: 템플릿 설정 + 링크 생성
- 입실자 페이지: 모바일 서명 (앱 없음)
- 모두싸인 API 연동 (전자서명법 준수)

**Stage 3: 고시원 네트워크 마켓플레이스**
- 만실 시 다른 GosiOps 원장님 고시원으로 cross-referral
- 외국인 레퍼럴 앰배서더 공식 파트너십
- "GosiOps 검증 고시원" 인증 뱃지

## MVP 데모 (토요일 시연 가능)

소프트웨어 없이 시연 가능합니다.

```
[시연 흐름]
1. 인터뷰 영상/사진: "원장님이 직접 이런 문제를 겪습니다"
2. 분쟁 케이스 실시간 입력 → AI가 조항 생성
3. 완성된 한/영 계약서 보여주기
4. "이 계약서가 다음 원장님에게도 적용됩니다"
```
